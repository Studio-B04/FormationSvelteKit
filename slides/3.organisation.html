3. Organisation des dossiers et des fichiers

--- layout: default transition: slide-up ---

Structure initiale du projet

Vous devriez obtenir un projet organisé comme ceci :

• Le dossier src contient les sources du projet, c'est à dire, tout ce qui constitue votre applicaiton.

• Le dossier static contient les élément publics du projet : les images, les polices, les facicones, etc.

• Le dossier .sveltekit est necessaire au build de l'application mais vous ne devez pas le modifier manuellement.

``` {*}{lines:false} my-app/ ├── .sveltekit/ │ └── ... ├── src/ │ ├── routes/ │ │ └── +page.svelte │ ├── lib/ │ │ └── index.js │ └── app.html ├── static/ ├── svelte.config.js ├── vite.config.js ├── package.json └── ... ```
--- layout: default transition: slide-up ---

Structure initiale du projet

Dossier / fichier Rôle
src/app.html Point d'entrée de votre application
src/lib/ Vos composants ou utilitaires partagés (accessibles avec l'alias $lib)
static/ Dossier public (images, favicon, etc.)
src/routes Dossier routes (gestion des URL de l'applciation)
src/routes/+page.svelte Page d’accueil de l'application
svelte.config.js Configuration de SvelteKit
vite.config.js Configuration de Vite

📂 static/

Tout ce qui se trouve dans le dossier static sera copié tel quel dans le dossier de build de l'application et sera accessible à la racine de votre application.

Vous pouvez donc y mettre des images, des polices, des fichiers CSS, etc.

``` {*}{lines:false} my-app/ └── static/ ├── favicon.png → accessible via /favicon.png └── images └── photo.jpg → accessible via /images/photos.jpg ```
