6. Le deploiement en production

--- layout: default transition: slide-up ---

Les adapters

Une fois votre développement terminé, vous allez vouloir deployer votre application sur votre serveur de production.

SvelteKit étant agnostique, il utilise des "adapters" pour builder votre application sur votre environement de production.

Chaque adapter traduit votre projet SvelteKit en une version compatible avec la cible choisie (serveur Node.js, CDN, Netlify, Vercel, etc.).

Pour utiliser un adapter, il faut l'installer via npm :

```bash npm install --save-dev @sveltejs/adapter-netlify ```
--- layout: default transition: slide-up ---

Les adapters

Un fois installé, il faut le configurer dans svelte.config.js :

```js import adapter from '@sveltejs/adapter-netlify'; export default { kit: { adapter: adapter() } }; ```

Maintenant, lorsque la commande npm run build sera exécutée, la fonction adapter sera appelée.

Essayons !

--- layout: default transition: slide-up ---

Les adapters

Lorsque la commande npm run build est lancée, l'adapter comoile tous le fichiers en tenant compte de vos différents fichies de configuration, et créé un nouveau dossier .netlify ainsi qu'un dossier build.

Il suffira alors de déposer le contenu du dossier build sur netlify !

--- layout: default transition: slide-up ---

Le CLI de Netlify

Netlify met à disposition des outils qui vous permettrons de deployer et gerer votre projet à distance en lignes de commande. Pour installer le CLI :

```bash {*}{lines:false} npm install netlify-cli -g netlify login netlify init netlify deploy --prod ```

Il suffira alors de créer un nouveau script dans le package.json pour deployer très facilement :

```json {*}{lines:false} { "scripts": { "deploy": "npm run test && npm run build && netlify deploy --prod", } } ```
